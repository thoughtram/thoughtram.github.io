Git Master Class

Git is a free and open source distributed version control system. We use it on our daily basis to be as productive as possible, especially when working in larger teams. Attending our Git Master Class gives you a deep understanding of how Git works and what tools it comes with to make working with code a lot easier.

Why take this training?

Have you ever wondered what the heck the HEAD is? And do you know how to properly do an interactive rebase without screwing everything up? And if you screwed it up, do you know how to restore things? Don't worry, you'll learn all this in this course. This course will make your average workday more productive. Every. Single. Day.

What you will learn

This is a quick run through the main topics which we will discuss. If you have questions not covered by the program, rest assured we don't leave any question unanswered!

Day 1

  • What is a distributed version control system (DVCS) anyway
  • How to install and configure Git
  • The anatomy of a Git commit
  • Working with the index
  • Everything about branches and tags
  • Fast-forward and Three-way merges
  • Other merge strategies and merge strategy options
  • How do I resolve conflicts
  • Everything about the different reset modes

Day 2

  • What is rebasing and how does it compare to merging?
  • Interactive rebasing: The most powerful Git feature
  • Squashing and splitting existing commits
  • Working with remotes
  • Shootout: rebase workflow vs merge workflow
  • How to cherry pick commits
  • The reflog: Never lose work again
  • Git internals

What you will take home

As a Git Master Class attendee you'll get a copy of all slides used to teach in the course, access to our private repositories with lots of examples, exercises and receipts and also to a closed mailing list for future support. In addition we provide you with great thoughtram brain stickers, so you can decorate your machine accordingly.
